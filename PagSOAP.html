SOAP
(Simple Object Access Protocol)

¿Qué es SOAP?

SOAP (Simple Object Access Protocol) es un protocolo de comunicación basado en XML diseñado para permitir el intercambio de información entre sistemas distribuidos en la web. Fue creado para superar los desafíos de interoperabilidad entre sistemas heterogéneos, permitiendo que aplicaciones de diferentes plataformas y lenguajes de programación puedan comunicarse entre sí de manera estandarizada.

Estructura del Mensaje

Envelope

Contenedor principal del mensaje

Header

Metadatos y configuración

Body

Contenido principal

Ejemplo SOAP

<?xml version="1.0" encoding="utf-8"?> <soap:Envelope xmlns:soap="http://www.w3.org/2003/05/soap-envelope/"> <soap:Header> <!-- Información adicional --> </soap:Header> <soap:Body> <!-- Contenido principal --> </soap:Body> </soap:Envelope>

Funcionamiento

Modelo Request/Response

Cliente envía solicitud, servidor devuelve respuesta

Transporte

Usa HTTP, HTTPS, SMTP o JMS

Serialización

Codificación de datos en XML

Roles

Sender

Envía mensajes

Receiver

Recibe mensajes

Intermediary

Reenvía mensajes

Ultimate

Destino final

Pros y Contras

Ventajas

  • Extensibilidad
  • Seguridad WS-*
  • Amplia adopción
Desventajas
  • Complejidad
  • Overhead XML
  • Rendimiento
  • Vulnerabilidades

Detalles Técnicos

  • WS-* Standards: Familia de especificaciones
  • WSDL: Lenguaje para describir servicios
  • UDDI: Directorio de servicios web
  • SOAP Fault: Manejo de errores
  • Attachments: MIME para archivos adjuntos
