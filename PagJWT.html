JWT
(JSON Web Token)

¿Qué es JWT?

JSON Web Token (JWT) es un estándar abierto (RFC 7519) que define un modo compacto y autocontenido para transmitir información segura entre distintas partes utilizando objetos JSON. Una de las características principales de JWT es que esta información puede ser verificada porque está firmada digitalmente.

Estructura de un JWT

Un JWT se compone de tres partes separadas por puntos (.) y serializadas en Base64:

Encabezado

Contiene el tipo de token y el algoritmo utilizado para la firma.

Carga útil

Almacena los datos relevantes del usuario, como su identidad o permisos.

Firma

Generada a partir del encabezado y la carga útil, utilizando una clave secreta.

Funcionamiento

Autenticación

El servidor genera un JWT al iniciar sesión y el cliente lo incluye en cada solicitud posterior.

Intercambio de Información

Útil para transferir datos entre servicios de manera segura.

Seguridad

La firma digital verifica la autenticidad del emisor y la integridad del contenido.

Pros y Contras

Ventajas
  • Compacto y autocontenido
  • Seguridad con firma digital
  • Flexibilidad en algoritmos
  • Escalabilidad para apps sin estado
Desventajas
  • Complejidad en manejo de claves
  • Vulnerable si clave es comprometida
  • No cifrado por defecto

Aplicaciones

  • Autenticación y Autorización: Gestión de acceso en apps web/móviles
  • Intercambio de Información Segura: Transferencia de datos entre servicios
  • Single Sign-On (SSO): Sistemas de inicio de sesión único
