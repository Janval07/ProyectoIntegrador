Docker

¿Qué es Docker?

Docker es una plataforma de código abierto que permite a los desarrolladores crear, probar e implementar aplicaciones dentro de contenedores. Estos contenedores son paquetes de software ligeros e independientes que encapsulan todo lo necesario para ejecutar una aplicación, incluyendo el código, bibliotecas, herramientas del sistema y ajustes.

Funcionamiento

Contenedores

Paquetes ligeros que aíslan aplicaciones usando el kernel del host

Imágenes

Plantillas que contienen código y dependencias para crear contenedores

Docker Engine

Motor central que gestiona contenedores, incluye Daemon, API y CLI

Contenedores vs Máquinas Virtuales

Característica Contenedores Máquinas Virtuales
Peso MBs (ligeros) GBs (pesados)
Arranque Segundos Minutos
Rendimiento Casi nativo Overhead por hipervisor
Aislamiento Procesos Sistema operativo completo

Aplicaciones

Desarrollo y Pruebas

Entornos consistentes y reproducibles

Implementación en la Nube

Despliegue en cualquier entorno compatible

CI/CD

Integración con herramientas de automatización

Ventajas

Ligereza

Menor consumo de recursos que VMs

Portabilidad

Funciona en cualquier entorno con Docker

Escalabilidad

Fácil ejecución de múltiples instancias

Simplificación

Desarrolladores se enfocan en el código

Ejemplo: WordPress con Docker

Despliegue simplificado de WordPress usando contenedores:

# Archivo docker-compose.yml version: '3' services: wordpress: image: wordpress:latest ports: - "8000:80" environment: WORDPRESS_DB_HOST: db WORDPRESS_DB_USER: exampleuser WORDPRESS_DB_PASSWORD: examplepass WORDPRESS_DB_NAME: exampledb volumes: - wordpress:/var/www/html db: image: mysql:5.7 environment: MYSQL_DATABASE: exampledb MYSQL_USER: exampleuser MYSQL_PASSWORD: examplepass MYSQL_RANDOM_ROOT_PASSWORD: '1' volumes: - db:/var/lib/mysql volumes: wordpress: db:

Ejecutar con: docker-compose up -d
