Kubernetes

¿Qué es Kubernetes?

Kubernetes (también conocido como K8s) es una plataforma de código abierto para automatizar la implementación, el escalado y la gestión de aplicaciones en contenedores. Originalmente desarrollado por Google y ahora mantenido por la Cloud Native Computing Foundation, Kubernetes se ha convertido en el estándar de facto para la orquestación de contenedores.

Funcionamiento

Pods

Unidad básica de implementación en Kubernetes, que puede contener uno o más contenedores

Nodos

Máquinas (físicas o virtuales) que ejecutan las cargas de trabajo en el clúster

Control Plane

Componentes que toman decisiones globales sobre el clúster y responden a eventos

Kubernetes vs Docker Swarm

Característica Kubernetes Docker Swarm
Complejidad Alta (más potente) Baja (más simple)
Escalabilidad Muy alta (5000+ nodos) Alta (hasta 1000 nodos)
Comunidad Muy grande y activa Limitada (Docker Inc.)
Casos de uso Producción a gran escala Pequeños despliegues
>

Ventajas

Escalabilidad

Escala aplicaciones horizontalmente con facilidad

Resiliencia

Auto-reparación y alta disponibilidad

Multi-nube

Funciona en cualquier entorno de nube

Ecosistema

Amplia gama de herramientas y extensiones

Componentes Clave

Pods

Unidades más pequeñas y básicas en el modelo de objetos de Kubernetes

Services

Abstracción que define un conjunto lógico de Pods y políticas para acceder a ellos

Volumes

Sistema de almacenamiento persistente para contenedores

Ejemplo: Despliegue Básico

Archivo YAML para desplegar una aplicación Nginx:

apiVersion: apps/v1 kind: Deployment metadata: name: nginx-deployment labels: app: nginx spec: replicas: 3 selector: matchLabels: app: nginx template: metadata: labels: app: nginx spec: containers: - name: nginx image: nginx:1.14.2 ports: - containerPort: 80 --- apiVersion: v1 kind: Service metadata: name: nginx-service spec: selector: app: nginx ports: - protocol: TCP port: 80 targetPort: 80 type: LoadBalancer

Aplicar con: kubectl apply -f deployment.yaml
