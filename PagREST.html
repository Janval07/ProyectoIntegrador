¿Qué es REST?

REST es un estilo de arquitectura de software que define cómo deben diseñarse las interfaces de programación de aplicaciones (APIs) para realizar una comunicación eficiente entre sistemas cliente-servidor. Fue propuesto por Roy Fielding en su tesis doctoral y se basa en el protocolo HTTP para la comunicación entre sistemas.

Características Principales

Cliente-Servidor

Separación clara entre cliente y servidor

Sin Estado

Cada solicitud contiene toda la información necesaria

Cacheable

Respuestas pueden almacenarse en caché

Interfaz Uniforme

URIs para recursos y formatos estándar

HATEOAS

Recursos incluyen enlaces relacionados

Funcionamiento

Verbos HTTP:
GET

Obtener un recurso

POST

Crear recurso

PUT

Actualizar recurso

DELETE

Eliminar recurso

Formatos de Datos:
JSON XML HTML Texto Plano
Escalabilidad:

Debido a su naturaleza sin estado, las APIs REST son más escalables que otras arquitecturas, ya que no requieren mantener sesiones en el servidor.

Pros y Contras

Ventajas
  • Flexibilidad en formatos
  • Escalabilidad horizontal
  • Sencillez de implementación
  • Amplia adopción
Desventajas
  • Complejidad en HATEOAS
  • Seguridad no incluida
  • Sobrecarga en metadatos

Aplicaciones

  • APIs Web: Estilo más común para APIs web
  • Integración de Sistemas: Conexión entre servicios
