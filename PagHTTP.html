HTTP / HTTPS

¿Qué es HTTP?

HTTP (Hypertext Transfer Protocol) es un protocolo de comunicación cliente-servidor que permite a los navegadores web solicitar y recibir datos de los servidores web. Es fundamental para la transferencia de información en Internet y se utiliza ampliamente en sitios web y API REST.

Funcionamiento de HTTP

1. Solicitudes y Respuestas

Los clientes (generalmente navegadores) envían solicitudes al servidor, y el servidor responde con los datos solicitados.

2. Métodos HTTP

GET

Recuperar datos

POST

Enviar datos

PUT

Actualizar datos

DELETE

Eliminar datos

3. Códigos de Estado

200 OK

Éxito

400 Bad Request

Error

404 Not Found

No encontrado

HTTP/2

  • Multiplexado: Múltiples mensajes simultáneos
  • Compresión de encabezados
  • Priorización de mensajes

Ejemplo HTTP

Solicitud:
GET /index.html HTTP/1.1 Host: www.example.com
Respuesta:
HTTP/1.1 200 OK Content-Type: text/html

¿Qué es HTTPS?

HTTPS (Hypertext Transfer Protocol Secure) es la versión segura de HTTP que utiliza cifrado SSL/TLS para proteger los datos transmitidos entre el cliente y el servidor.

Proceso HTTPS

1. Solicitud SSL

El navegador solicita el certificado SSL del servidor

2. Verificación

El navegador verifica la autenticidad del certificado

3. Cifrado

Se establece conexión cifrada con SSL/TLS

Ventajas/Desventajas

Ventajas HTTP
  • Fácil implementación
  • Soporte universal
Desventajas
  • Datos no cifrados
  • Inseguro para datos sensibles
